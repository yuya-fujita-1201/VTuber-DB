# E2Eテスト計画書

## 1. テスト範囲

- **フロントエンド**: 主要なユーザーフロー（検索、詳細表示、タグクリック）
- **API**: 主要なエンドポイント（検索、詳細取得）

## 2. テストケース

### 2.1. フロントエンド

| ID | テストケース | 手順 | 期待される結果 |
|---|---|---|---|
| FE-01 | タグ検索 | 1. 検索ページを開く<br>2. 「ゲーム実況」タグを選択<br>3. 検索ボタンをクリック | 1. 検索結果に「ゲーム実況」タグを持つVTuberが表示される<br>2. 「次に辿る候補タグ」に「FPS」「ホラーゲーム」などが表示される |
| FE-02 | 候補タグ追加 | 1. FE-01の後、「FPS」をクリック | 1. 「FPS」タグが検索条件に追加される<br>2. 検索が自動的に実行される |
| FE-03 | VTuber詳細表示 | 1. 検索結果からVTuberを1人クリック | 1. VTuber詳細ページに遷移する<br>2. タグにscoreと根拠が表示される<br>3. 「似ているVTuber」が表示される |

### 2.2. API

| ID | テストケース | リクエスト | 期待される結果 |
|---|---|---|---|
| API-01 | タグ階層検索 | `GET /api/search?tags=1` (ID:1が「エンタメ」) | `data`に「ゲーム実況」や「歌」タグを持つVTuberが含まれる |
| API-02 | 探索支援 | `GET /api/search?tags=2` (ID:2が「ゲーム実況」) | `suggested_tags`に「FPS」や「ホラーゲーム」が含まれる |
| API-03 | 詳細API拡張 | `GET /api/vtubers/1` | レスポンスに`tags.evidence`と`similar_vtubers`が含まれる |

## 3. テスト環境

- **ブラウザ**: Google Chrome 最新版
- **ツール**: Postman (APIテスト用)
